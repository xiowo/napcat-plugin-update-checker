# 🦊 插件更新检测 (napcat-plugin-update-checker)



这是一个专为 **NapCat** 框架设计的强大插件更新检测与管理工具。它能够自动监控已安装插件的版本，支持多源扩展、定时提醒、自动更新以及完善的 WebUI 管理界面，让您的 NapCat 插件始终保持最新状态。

---

## ✨ 核心功能

- 🛡️ **智能检测**：自动对比已安装插件与商店索引的版本差异。
- 🌐 **多源支持**：内置社区插件库，并支持添加自定义第三方插件源。
- ⏰ **定时巡检**：自定义检查间隔，无需人工干预即可在第一时间掌握更新动态。
- 🚀 **镜像加速**：内置数十个 GitHub 加速镜像，支持 Raw 索引与下载包的双重测速与选择。
- 🔄 **自动更新**：支持 `仅通知` 与 `全自动更新` 两种模式，并配有灵活的白名单/黑名单机制。
- 📊 **可视化面板**：提供精美的 WebUI 仪表盘，支持一键测速、配置修改及插件管理。
- 📦 **便捷安装**：支持从应用商店在线安装，或通过本地 ZIP/文件夹进行导入。
- 🎨 **主题定制**：内置多套 UI 主题预设，支持自定义配色。

## 🛠️ 安装指南

### 1. 自动安装
~~在 NapCat 的插件市场中搜索 `插件更新检测` 并点击安装。~~

### 2. 手动安装
1. 下载最新版本的 [Release](https://github.com/xiowo/napcat-plugin-update-checker/releases) 压缩包。
2. 将其解压到 NapCat 的 `plugins` 目录下。
3. 重启 NapCat 即可。

### 3. 开发者编译
如果您希望自行构建项目，请确保已安装 Node.js 环境：
```bash
# 克隆仓库
git clone https://github.com/xiowo/napcat-plugin-update-checker.git
cd napcat-plugin-update-checker

# 安装依赖
pnpm install

# 编译项目
pnpm run build
```

## 💬 聊天指令

插件默认监听前缀为 `#更新` 的指令（可在配置中修改）：

| 指令 | 说明 | 权限 |
| :--- | :--- | :--- |
| `#更新 帮助` | 显示所有可用指令列表 | 仅主人 |
| `#更新 状态` | 查看当前检查器运行状态、上次检查时间等 | 仅主人 |
| `#更新 检查` | 立即执行一次全局更新检测 | 仅主人 |
| `#更新 全部` | 一键更新所有发现新版本的插件 | 仅主人 |

> 权限规则：仅 `masterQQ` 中配置的 QQ 可以执行命令（支持英文逗号分隔多个 QQ）。

## ⚙️ 配置说明

可在 WebUI 扩展页面中修改，核心配置项如下：

| 配置项（WebUI） | 默认值 | 描述 |
| :--- | :--- | :--- |
| `启用插件` | `true` | 是否启用此插件功能（群内命令） |
| `命令前缀` | `#更新` | 触发命令的前缀 |
| `冷却时间（秒）` | `60` | 同一命令请求冷却时间，`0` 表示不限制 |
| `主人QQ` | - | 主人 QQ，多个用英文逗号分隔；设置后仅主人可执行管理命令 |
| `黑名单用户` | - | 黑名单 QQ 列表（支持逗号分隔输入），黑名单用户无命令权限 |
| `更新模式` | `仅通知` | `仅通知` 或 `自动更新` |
| `启用定时检查` | `true` | 是否启用定时检查插件更新 |
| `检查间隔（分钟）` | `30` | 定时检查更新的间隔时间（最小 1 分钟） |
| `通知群聊` | - | 接收更新通知的群列表 |
| `通知用户` | - | 接收更新通知的用户（私聊）列表 |
| `自动更新插件` | - | 自动更新插件列表（为空表示不指定） |
| `黑名单插件` | - | 黑名单插件列表（不会参与自动更新） |
| `商店源管理` | - | 插件商店源列表，支持多源配置与启停 |

## 🖥️ WebUI 扩展页面

本插件深度集成 NapCat WebUI，注册了标识为 `update-checker-dashboard` 的扩展页面。

**您可以在此处进行：**
- **仪表盘总览**：查看当前版本、运行日志与更新统计。
- **镜像测速**：一键测试全球数十个镜像源的延迟，并自动保存最优节点。
- **商店管理**：添加或删除第三方插件源，实时浏览在线插件。
- **配置同步**：图形化修改所有配置项，即刻生效。

## 📂 项目结构

```text
src/
├── index.ts              # 核心入口，负责生命周期与路由注册
├── config.ts             # 默认配置定义与 WebUI Schema 构建
├── types.ts              # 严格的 TypeScript 类型定义
├── core/
│   └── state.ts          # 全局状态管理与持久化存储

│   └── scheduler.ts      # 定时任务调度与多渠道消息推送
└── webui/
    └── dashboard.html    # WebUI 扩展页面前端代码
```
